     Migliorie individuali per il progetto Music Release Artists

     Context

     Il progetto e' un monorepo ben strutturato (AdonisJS v7 backend + React 19
     frontend) per tracciare le uscite musicali degli artisti seguiti su
     Spotify. Il codice e' pulito e funzionale, ma ci sono diverse aree
     migliorabili a livello di qualita', manutenibilita' e robustezza.

     ---
     1. Rimuovere codice morto e dipendenze inutilizzate

     Frontend:
     - @tuyau/client (frontend/package.json:16) ‚ÄĒ installato ma mai importato.
     Il frontend usa un wrapper fetch manuale (src/lib/api.ts). Rimuovere con
     npm uninstall @tuyau/client.
     - PrefetchLinkToDashboard e PrefetchLinkToSettings
     (src/components/ui/prefetch-link.tsx:76-98) ‚ÄĒ esportati ma mai usati in
     nessun componente. Rimuovere.
     - await Promise.resolve() (src/pages/reset-password.tsx:22) ‚ÄĒ no-op,
     rimuovere la riga.

     2. Usare il client Tuyau tipizzato al posto del fetch manuale

     Il backend genera un typed client registry con @tuyau/core (configurato in
     adonisrc.ts), ma il frontend non lo usa ‚ÄĒ tutto l'accesso API e' in
     src/lib/api.ts con un wrapper fetch non tipizzato. Due opzioni:
     - Opzione A: Integrare @tuyau/client nel frontend per avere type-safety
     end-to-end.
     - Opzione B: Rimuovere @tuyau/core dal backend se non serve (piu'
     semplice).

     3. Aggiungere test frontend (Vitest)

     Non esiste nessun framework di test nel frontend. Aggiungere:
     - vitest + @testing-library/react + jsdom
     - Test per hooks critici (useAuth, useReleases)
     - Test per componenti UI (FilterBar, ReleaseCard)

     4. Migliorare la copertura dei test backend

     I test esistenti (5 functional + 1 unit) coprono solo casi base e scenari
     di errore auth. Mancano:
     - Test per il flusso Spotify OAuth (redirect, callback, disconnect)
     - Test per il sync artisti e le live releases
     - Test per notification settings (CRUD)
     - Test per il comando send_release_digest

     5. Aggiungere CI/CD (GitHub Actions)

     Non esiste nessuna pipeline CI/CD. Creare .github/workflows/ci.yml:
     - Lint + typecheck backend e frontend
     - Test backend (node ace test)
     - Test frontend (quando aggiunto Vitest)
     - Build Docker images

     6. Rate limiting sulle rotte di autenticazione

     Solo /artists/sync ha rate limiting (5 req/min). Mancano:
     - /auth/login ‚ÄĒ protezione contro brute-force
     - /auth/forgot-password ‚ÄĒ protezione contro abuse
     - /auth/verify-email/resend ‚ÄĒ protezione contro spam

     File da modificare: start/routes.ts, start/limiter.ts

     7. Scheduler per il digest command

     Il comando send_release_digest esiste ma non c'e' nessun cron/scheduler
     configurato. Opzioni:
     - Usare @adonisjs/scheduler o un cron di sistema
     - Oppure aggiungere un servizio cron nel docker-compose.yml

     8. Centralizzare i colori custom in Tailwind

     I colori del brand (#aa3bff, #08060d, #6b6375, #e5e4e7, #f4f3ec) sono
     hardcoded nelle classi Tailwind in molti componenti. Creare un file app.css
      con custom theme tokens (Tailwind v4 @theme) per evitare duplicazione e
     facilitare il rebranding.

     9. Aggiungere git hooks (lint-staged + husky)

     Non ci sono git hooks. Aggiungere:
     - husky per i git hooks
     - lint-staged per eseguire ESLint e Prettier solo sui file modificati prima
      del commit

     10. Healthcheck nel Dockerfile backend

     Il Dockerfile del backend (backend/Dockerfile) non ha un HEALTHCHECK. Il
     docker-compose.yml lo definisce, ma il Dockerfile standalone no.
     Aggiungere:
     HEALTHCHECK CMD wget -qO- http://localhost:3333/health || exit 1
     (Richiede anche un endpoint /health nel backend se non esiste.)

     11. Internazionalizzazione (i18n)

     Le stringhe UI sono hardcoded in italiano nel frontend e nelle email del
     backend. Se il progetto dovra' supportare altre lingue, conviene introdurre
      un sistema i18n fin da subito (es. react-i18next). Se resta solo italiano,
      non serve.

# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Project Overview

Music release artists app with a monorepo structure: AdonisJS v7 API backend + React/Vite frontend.

## Commands

### Backend (run from `backend/`)
- **Dev server:** `node ace serve --hmr`
- **Build:** `node ace build`
- **Lint:** `eslint .`
- **Format:** `prettier --write .`
- **Typecheck:** `tsc --noEmit`
- **Run tests:** `node ace test`
- **Run single test suite:** `node ace test --suite unit` or `--suite functional`
- **Run migrations:** `node ace migration:run`
- **Create migration:** `node ace make:migration <name>`
- **Create controller:** `node ace make:controller <name>`

### Frontend (run from `frontend/`)
- **Dev server:** `npm run dev` (Vite)
- **Build:** `tsc -b && vite build`
- **Lint:** `eslint .`

## Architecture

### Backend — AdonisJS v7

- **Database:** SQLite via better-sqlite3 (file at `tmp/db.sqlite3`). Postgres/MySQL configs are commented out in `config/database.ts`.
- **Auth:** Token-based (access tokens) as the default guard (`api`), with a session-based guard (`web`) also configured. See `config/auth.ts`.
- **Schema generation:** Lucid models extend auto-generated schema classes from `database/schema.ts` (regenerated by `migration:run`). The `User` model composes `UserSchema` + `withAuthFinder`.
- **API responses:** A custom `ApiSerializer` in `providers/api_provider.ts` wraps all responses in `{ data: ... }`. Controllers use `ctx.serialize()` to produce consistent output.
- **Transformers:** Used to control which model fields are exposed (e.g., `UserTransformer` picks specific fields including computed `initials`).
- **Validators:** VineJS validators in `app/validators/`. Signup validates uniqueness against the DB.
- **Tuyau:** `@tuyau/core` generates a typed client registry from routes (see `adonisrc.ts` hooks).
- **Module imports:** Use `#` subpath imports (e.g., `#models/user`, `#controllers/*`), defined in `package.json` `imports` field.

### Frontend — React 19 + Vite 8

- Uses React Compiler via `@vitejs/plugin-react` + `@rolldown/plugin-babel` with `reactCompilerPreset`.
- Currently a starter template — main app code is in `src/App.tsx`.

### API Routes (prefixed `/api/v1`)

- `POST /auth/signup` — create account
- `POST /auth/login` — get access token
- `POST /auth/logout` — revoke token (auth required)
- `GET /account/profile` — get current user profile (auth required)
